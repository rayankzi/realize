# How Recruiters Rank Your Python Projects for Quant

**Topic/Category:** Quant Finance / Career Advice / Python Projects
**Primary Goal of the Reel:** To give aspiring quant traders a clear, tiered ranking of Python projects — showing which ones actually impress recruiters and which ones quietly hurt your application.

## Executive Summary

This reel provides a brutally honest breakdown of how quant recruiters evaluate Python projects on your CV. The creator categorizes projects into three tiers: Tier 1 projects that demonstrate genuine quant thinking and land interviews, Tier 2 projects that show coding ability but lack analytical depth, and Tier 3 projects that actively damage your application. The core message is that one well-built Tier 1 project outweighs ten mediocre ones, and the standard should be set early in your career.

## Core Insights & Takeaways

- **Insight 1:** Quant recruiters care about **quant thinking**, not just coding ability. The distinction between building something functional and building something that demonstrates financial reasoning is everything.
- **Insight 2:** ML-based stock prediction models are a red flag — they signal that you don't yet understand why machine learning struggles with financial time series data.
- **Insight 3:** Any project you cannot fully defend and explain under questioning will cost you more than having a blank CV. Recruiters interrogate every line item.
- **Insight 4:** One properly built Tier 1 project beats ten Tier 2 projects every single time. Quality over quantity is the rule.

## Narrative & Step-by-Step Breakdown

**Beginning — The Hook:**
The creator opens with a direct, authoritative statement framing the reel as insider knowledge: "Not all Python projects are equal for quant. Here's exactly how recruiters rank what you build."

**Middle — The Tier System:**

### Tier 1: Gets You Interviews
1. **Momentum backtest from scratch** — No libraries doing the heavy lifting. Clean code, honest performance analysis, and a write-up explaining what you measured and why it matters.
2. **Monte Carlo simulation** — Models portfolio outcomes or prices a simple instrument under uncertainty.
3. **Volatility targeting model** — Scales position size inversely to realised volatility using real market data.

### Tier 2: Gets You Noticed But Not Hired
1. **Stock price prediction model using ML** — Shows Python ability but signals a misunderstanding of ML's limitations in financial time series.
2. **Data dashboard pulling API data** — Visually impressive but lacks analytical depth underneath.
3. **Pairs trading script** — Finds correlated assets but ignores transaction costs, slippage, and regime breaks.

### Tier 3: Hurts Your Application
1. **Unmodified YouTube tutorial projects** — Copied without any original contribution.
2. **Deep learning on stock returns labelled as "algorithmic trading"** — Misrepresents capabilities.
3. **Any project you can't fully explain** — If you can't defend every design decision, it's a liability.

**End — The Standard:**
The creator concludes with a sharp takeaway: "One Tier 1 project built properly beats ten Tier 2 projects every time. That's the standard. Set it in first year."
